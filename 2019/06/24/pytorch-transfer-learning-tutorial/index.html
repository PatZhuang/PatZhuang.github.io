Transfer Learning Tutorial

发表于 2019-06-24

本文是 TRANSFER LEARNING TUTORIAL 的学习笔记。

本文将展示如何使用迁移学习来训练你的网络。有关迁移学习的更多内容见 cs231n notes

引用 CS231n 的说法：

In practice, very few people train an entire Convolutional Network from scratch (with random initialization), because it is relatively rare to have a dataset of sufficient size. Instead, it is common to pretrain a ConvNet on a very large dataset (e.g. ImageNet, which contains 1.2 million images with 1000 categories), and then use the ConvNet either as an initialization or a fixed feature extractor for the task of interest.

在实际使用中，很少有人从头训练一个卷积神经网络（使用随机初始化的权重），因为我们很难获得足够大的数据集。常见的做法是在一个很大的数据集（比如 ImageNet, 它有 1000 类 共 120 万张图片）上预训练一个卷积神经网络，然后使用这个网络的权重作为我们的初始化权重，或者将这个网络作为固定的特征提取器。

需要使用迁移学习的两个主要场景如下：

  • 对卷积神经网络做微调：使用预训练的网络而不是随机权重来初始化我们的网络，之后再进行常规的训练过程。
  • 把卷积神经网络作为固定特征提取器：把卷积神经网络除最后一层全连接层以外的所有层的权重冻结（不更新），再使用一个新的全连接层来替代最后一层，并将其权重随机初始化。

CS231n notes 对 “Fine-tuning the Convnet” 的说法是：

(It) is to not only replace and retrain the classifier on top of the ConvNet on the new dataset, but to also fine-tune the weights of the pretrained network by continuing the backpropagation.

即对预训练模型微调的方法是：替换分类器（最后一层全连接层），并继续对执行反向传播来调整权重。

这个过程既可以调整所有层的权重（如本例），也可以只训练靠后的（靠近输出）的层而固定一些靠前的层的权重来降低过拟合的风险。后者的原因是：可以观察到靠前的层提取到的特征是更宽泛的特征（比如边缘或颜色提取），而靠后的层提取到的特征会更针对数据集。

CS231n notes 对 “ConvNet as fixed feature extractor” 的说法是：

Take a ConvNet pretrained on ImageNet, remove the last fully-connected layer (this layer’s outputs are the 1000 class scores for a different task like ImageNet), then treat the rest of the ConvNet as a fixed feature extractor for the new dataset.

和微调一样，将预训练模型作为特征提取器时，也是替换最后一层全连接层。

对于 AlexNet，这样的结果是将在最后一层之前计算出一个 4069 维的向量，这个向量也被称为 CNN codes. 对所有的图像都提取出这些特征后，再训练一个线性分类器（线性 SVM 或者 softmax，本例使用的是一个简单的全连接层）就可以。

When and how to fine-tune?

CS231n notes 给出了以下四个主要场景：

  1. 新的数据集较小，而且和模型对应的原数据集相似。

    在这种情况下，考虑到过拟合问题，对原模型进行微调（fine-tune）是不太合适的。而由于两个数据集相似，我们可以认为该模型的高层特征与我们的数据集也是相关的。因此，最好的做法是对 CNN codes 训练一个线性分类器。

  2. 新的数据集很大，而且和模型对应的原数据集相似。

    在数据集很大的情况下，可以对原模型进行微调而不用太担心会造成过拟合问题。

  3. 新的数据集较小，而且和模型对应的原数据集有很大差异。

    同样的，对于小数据集来说，较好的做法是训练一个线性分类器。但是由于我们的数据集和原始数据集差异较大，模型高层特征和原数据集有一定的相关性。因此最好的做法是从模型较早的某层开始训练一个线性分类器。

  4. 新的数据集很大，而且和模型对应的原数据集有很大差异。

    在这种情况下，从头训练一个模型是可以的。但是在实际应用中，使用预训练模型的权重并对整个模型进行微调通常仍然是有益的。

Practical advice

CS231n notes 给出了两个使用建议：

  • 关于预训练模型的限制：

    如果要使用预训练模型，那么对新数据集使用的模型架构会有所限制。例如说，你不能随意地从预训练模型中拆除某个卷积层。不过某些简单的更改是可以做到的：由于参数共享（parameter sharing），你可以对有不同空间大小（spatial size）的图片执行训练。这是因为卷积层和池化层的前向传播运算无关图片空间大小，而全连接层可以相应替换成卷积层来规避这个问题。

    这里比较 tricky，参考 CS231n 课程笔记翻译

  • 关于学习率：

    比起训练分类器，在微调模型的时候一般会使用更小的学习率。这是因为我们认为预训练模型的权重已经相当不错，而我们不想过快和过分地调整这些权重（特别是当顶层的分类器是随机初始化的时候）。
